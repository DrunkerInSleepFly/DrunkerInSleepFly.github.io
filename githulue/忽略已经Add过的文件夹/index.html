忽略文件夹

解决添加了 .gitignore 文件路径还是没有被忽略的问题
问题描述：.gitignore 中有忽略文件路径，但是git add . 时依然被添加进去，目录路径没问题的
原因： 这个文件是在我提交了好多次后才发现老是有些logs或run被 git add . 进去
后来发现了，想忽略掉，发现 .gitignore 没有起作用，原因是是 文件已存在跟踪列表

解决方法：

1.git rm –cached “文件路径”
2.如果出现 not removing ‘game/logs’ recursively without -r
3.执行 git rm -r –cached “文件路径”
4.最后记得在 .gitignore 确认添加忽略文件路径
5.当服务器 git pull 时提示服务器有变动 一概使用以下命令git reset –hard 强制覆盖 然后再 pull git
